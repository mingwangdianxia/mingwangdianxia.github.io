hugo的简单介绍与使用

Hogo

一个快速搭建前端静态页面的框架

安装

要想使用这个hugo工具，需要先在你的服务器或者自己的电脑上安装这个软件

使用这个软件可以快速的生成静态的html代码，你拿到这个静态代码后放到nginx或其他web服务上就能访问你的项目。

下面介绍一下如何安装：

首先奉上windows安装的官方教程：https://gohugo.io/installation/windows/

看官方的教程发现有两样新东西：Prebuilt binaries（hugo安装包）、Package managers（包管理工具）。

Prebuilt binaries：hugo安装包，这个没什么悬念，下载安装就可以了，介绍上说是一些跟操作系统有关的吧啦吧啦；

Package managers：包管理工具，这个在windows上很少能听到，因为windows上安装软件基本都是双击，然后选择一个安装目录，一直下一步就行，很少有用到包管理这个概念， 官方还介绍了Chocolatey，Scoop，Docker这几个工具，就用排在最前面的来。

下面是自己的实践

1、根据自己电脑配置下载对应的安装包：

安装包地址： https://github.com/gohugoio/hugo/releases/tag/v0.105.0

2、安装包管理工具CHOCOLATEY

官网：https://chocolatey.org/

这个工具是类似linux中的apt-get、unix的brew，是以命令行的形式安装一些软件的工具。

用管理员身份打开powerShell，然后输入下面命令回车即可安装成功

Set-ExecutionPolicy Bypass -Scope Process -Force; [System.Net.ServicePointManager]::SecurityProtocol = [System.Net.ServicePointManager]::SecurityProtocol -bor 3072; iex ((New-Object System.Net.WebClient).DownloadString('https://community.chocolatey.org/install.ps1'))

下面是执行过程中的截图

3、 安装hugo

在cmd中执行下面的命令即可

choco install hugo-extended

4、查看是否安装成功

查看hugo的版本号

hugo version

快速开始

安装好hugo之后，便可以快速开始搭建一个简单的前端网站，基本上按照官方教程执行就能正常建站，我在这一步没有遇到问题，就不贴实践过程了

官方教程：https://gohugo.io/getting-started/quick-start/

基础使用

  • 查看是否安装成功
hugo version
  • 查看帮助
hugo help

# 查看二级命令的帮助
hugo command --help
hugo server --help
  • draft(草稿)、future(将来)、expired(过期)
# hugo可以针对上面三种类型的文章、博客进行管理，并控制是否发布

hugo --buildDrafts    # or -D
hugo --buildExpired   # or -E
hugo --buildFuture    # or -F
  • 运行项目

执行下面命令会在内存中构建网站，以便从浏览器中更好的看到，默认的网站地址是：http://localhost:1313

并且不需要任何操作，在更改了网站内容之后，只要刷新浏览器就能够看到最新内容

hugo server
  • 打包项目
hugo
